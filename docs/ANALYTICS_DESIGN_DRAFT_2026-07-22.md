# 분석·수익화 기능 설계 초안 (2026-07-22, 검수 전 DRAFT)

> 소유자 지시로 작성한 초안. **소유자는 "범위 좁히기"를 싫어함** → 최대주의 설계.
> GPT 검수 + 소유자 승인 전까지 구현하지 않음.

## 목표 (소유자 의중)
1. **최대한 상세한 사용 통계** — 니즈 파악(뭘 개발/삭제할지) + 수익화(맞춤광고).
2. **GA(방문 통계) + 자체 대시보드(내부 사용 통계) 둘 다.**
3. **학번 하나로** 학년·학과·시간표를 자동 로드(편의) + 학과·학년 기반 맞춤광고.

## 데이터 수집
- **학번(8자리)** = 가입/인증 시 받음(이미 중복가입 방지용 HMAC 처리 중). 파싱:
  `[입학년도4][학과코드2][ㄱㄴㄷ순번2]` → **학년**(입학년도+현재연도)·**학과**(코드표) 도출.
  - 학과코드표(13개, A/B군) 확보됨 → `app/lib/hakbeon.js` 유틸에 내장(학번→{학년,학과,군}).
  - ⚠️ 2025~ 학생증엔 "초등교육학과" 통일 → 학과는 **학번 코드로만** 도출(학생증 표면 X).
- **편의**: 학번 넣으면 학년·학과·**이번학기 시간표 자동 채움**("학번만 넣으면 다 불러와져요").
- **행동 이벤트**(first-party): 버튼 클릭·화면 진입·기능 사용을 우리 서버로 집계.

## 동의 모델 (PIPA 정렬 — 가입 약관, 상시배너 없음)
- **필수(최소)**: 학번(중복가입·인증·서비스), "통계·서비스개선 목적 수집" 고지.
- **선택 옵트인**: "학번(학과·학년) 기반 맞춤광고에 동의?" → 동의=맞춤, 미동의=**일반광고**(앱은 그대로 씀).
- 학번 원문은 광고주에 안 넘김 — 파생 **학과·학년 세그먼트**로 first-party 타겟만.

## A. GA (방문 통계)
- GA4 스니펫 삽입. 방문자·세션·페이지뷰·유입·기기·체류·실시간.
- 동의 상태 연동: 옵트인 시 개인화 신호, 미동의 시 익명 집계만.

## B. 자체 대시보드 (`/admin`, 운영자 전용 — actor_role_check operator+)
**모든 지표를 학년·학과로 세그먼트해서 봄**(핵심: 1학년 vs 4학년, 국어과 vs 영어과 니즈).

- **회원**: 총가입/일·주 신규추이, 인증완료·대기, 학교이메일 vs 외부(015 후), DAU/WAU/MAU, 제재수, **학년·학과 분포**.
- **AI 지도안**: 생성건수(약안/세안), 하루 원가·예산소진율, 실패·잘림율, **학년·학과별 사용**.
- **시간표**: 생성 수·마법사 사용·저장수·학점계산기 사용.
- **게시판**: 게시판별 글/댓글/조회/**진입클릭**, 추천·스크랩·신고, 활성 랭킹.
- **버튼/화면**: **모든 주요 버튼·화면 진입 클릭 카운트**, 이탈 지점.
- **강의검색·정보**: 검색횟수·초성비율·인기검색어(익명)·급식/공지/캘린더/e-Class 클릭.
- **실습·기타**: 실습모드·실습학교설정·강의평가작성·SR 발행/소비.

## 수익화 (광고)
- **학과·학년 기반 first-party 스폰서 슬롯**(AdSense가 못하는 정밀 타겟) — 옵트인 사용자.
  예: 4학년→임용학원, 1학년→새내기, 국어과엔 영어광고 안 띄움.
- GA/AdSense = 익명 방문·일반광고 보조 축.

## 아키텍처 (분리 원칙)
- `app/lib/hakbeon.js` — 학번 파싱(학년·학과·군), 순수함수·테스트.
- 이벤트 수집: 클라 → `POST /api/track`(경량) → `private.usage_events` 또는 집계 카운터.
  개인 미결합 원칙: (이벤트, 날짜, [옵트인 시 학년·학과 세그먼트])만. 실명·학번 원문 없음.
- 대시보드: `/admin/analytics` — 운영자 전용, Supabase 집계 쿼리 + 카운터 읽기.
- GA: 동의 상태에 따라 로드하는 클라 스니펫.

## 나의 경계 (소유자에게 고지)
- 집계 통계는 최대한 상세히. **개인 단위 프로파일을 동의 없이 광고주에 넘기지 않음.**
- 맞춤광고 옵트인·학번 원문 미전송. 이 선 안에서 최대주의.

## ── 확정 설계 (GPT 검수 P-20260722 반영, DESIGN_DIRECTION=APPROVED / 범위축소 NO) ──

**대원칙: 데이터 3층 분리 + 목적별 독립 동의.** 광고 기능은 전부 구현하되 **flag 뒤에
휴면**시켜, 사업자등록 되는 순간 on/off로 켠다(소유자 지시).

### 층·flag 구조 (features.js)
| 계층 | 내용 | flag | 활성 조건 |
|---|---|---|---|
| 필수 | 운영·보안·과금 최소 계측 | (flag 없음, 상시) | — |
| 편의 | 학번→학년·학과·권장시간표 자동로드 | `hakbeonAutofill` | 학번 인증단계 수집 후 |
| 선택1 | 상세 제품분석 + /admin 대시보드 | `productAnalytics` | MUST 반영 후(사업자 불요) |
| 선택2 | **학과·학년 맞춤 스폰서 광고(전부 구현, 휴면)** | `targetedAds` | **사업자등록 + MUST 후 켬** |

### 데이터 수집 (GPT MUST 반영)
- **학번은 "학생 인증 단계"에서만 수집**(전원 가입 시 X — 로그인 개방됨). HMAC(중복가입)과
  analytics/ad 식별자는 **키·목적 분리**. 학번 원문은 로그·URL·GA4·광고요청 미기록.
- 학번 파생: `entry_year`(확정)·`entry_department`(코드확정)·`expected_grade`(계산)·
  `current_grade`(사용자 확인·수정)·`current_major`(사용자확인). 학과=입학당시, 복수전공 별도.
  코드표는 **입학연도별 버전**(`app/lib/hakbeon.js`), 미등록·비정상 임의추정 금지.
- 시간표 = "**권장 시간표 초안**"(학번만으론 개인 수강변경 모름 → 사용자 확정).
- 성별·나이 = 학번추론 금지. 필요 시 각각 선택+"응답 안 함".

### 동의 (약관형, 상시배너 X)
- 필수: 학번(인증·중복가입) + "통계·서비스개선" 고지.
- **기본 OFF 독립 선택 2개**: `[선택] 상세 이용통계` / `[선택] 학과·학년 맞춤 스폰서 광고`.
  상세분석 동의가 광고 동의로 자동승격 안 됨. consent version·동의시각·철회시각 저장. 거부해도
  핵심서비스+일반광고. **맞춤광고는 만 18세 이상 확인자만**(그 외 일반광고).

### 이벤트 수집 (registry 방식, pseudonymous)
- 허용 이벤트만: `screen_view`·`feature_start`·`feature_complete`·`button_click`·
  `search_submitted`·`error`·`sponsor_impression`·`sponsor_click`.
- 금지: keystroke·자유서술·글/댓글 내용·인증서류·이메일/학번/회원ID·URL 식별자·원문검색어
  무제한·DM/신고/상담. `POST /api/track` allowlist schema 검증.
- 3파이프라인: **미동의**=즉시 counter만(ID·세그먼트 저장X) / **분석동의**=무작위
  `analytics_subject_id`+학과학년 segment, 원시 90일 후 집계만 / **광고동의**=학과학년 segment로
  광고선택(행동 이벤트는 광고선택에 미사용).
- **k-익명**: 대시보드 셀≥5, 맞춤광고 segment≥20, 광고주 보고≥10·24h지연 + complementary suppression.

### /admin/analytics (운영자 전용)
- actor_role_check operator+ + 서버측 권한감사·기초테이블 직접접근 차단·조회 audit·원시
  drill-down 없음·CSV 원시 export 기본금지·소수셀 suppression.
- 지표: 회원·AI원가·시간표·게시판·버튼·검색·실습을 **학년·학과 세그먼트**로. +GPT 추가:
  전환퍼널·D1/D7/D30·기능채택속도·무결과검색·지도안 중단률/SR부족·**기능별 원가·수익·유지율**.

### 광고 (targetedAds flag 뒤, 전부 구현·휴면)
- 옵트인=우리 서버가 학과·학년으로 **first-party 스폰서 슬롯** 선택. 미옵트인=일반 자체 스폰서
  or 비개인화 Google. **Google 요청에 학번·HMAC·회원ID·학과·학년·검색어 미전송.**
- 네이티브=AdMob/Ad Manager(AdSense 아님). 광고·후원 표시, 시스템버튼 오인·UI겹침 금지,
  광고주 픽셀·SDK·외부링크 학과학년 query 금지, Google 클릭에 자체보상 안 붙임. 리워드=Google
  공식 Rewarded Ads(SR 현금화·양도불가 유지). 광고주엔 집계만.
- GA4는 `analytics_storage/ad_storage/ad_user_data/ad_personalization` Consent Mode 제어,
  학과·학년·학번 미전송, PII redaction. 처리방침·Play Data Safety·App Privacy Label 실제 일치.

### GA(방문통계)
- GA4, 익명 방문·페이지·유입만. Consent Mode 연동. 개인식별·학과·학년 미전송.

### 구현 순서 (제안)
1. `app/lib/hakbeon.js`(학번 파싱, 순수함수+테스트) + 학번을 인증단계에서 수집·파생 저장.
2. 동의 3층(약관·설정 UI + consent 저장) + `/api/track`(registry) + `usage_events`/카운터.
3. `/admin/analytics` 대시보드 + 집계·k-익명·audit. ← 여기까지 `productAnalytics` flag로 활성(사업자 불요).
4. 광고: 스폰서 슬롯·타겟·Consent Mode·리워드 — **전부 구현 후 `targetedAds` flag OFF로 휴면**. 사업자등록 시 ON.

각 단계 GPT 검수 → 적용/배포. 큰 기능이라 writing-plans로 단계별 계획 잡아 진행.
