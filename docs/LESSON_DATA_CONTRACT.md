# 지도안 생성기 데이터 계약 (v1)

> 로컬 GPT 가 만드는 CSV 를 앱이 그대로 삼킬 수 있게 **형식을 먼저 고정한다.**
> 데이터가 다 모인 뒤에 형식을 정하면 전부 다시 만들어야 하므로, 아직
> 비어 있는 지금 못 박는다.
>
> 대상 파일: `로컬GPT_작업지시서_2차.md` 의 §1 · §2 · §5 산출물
> 소비하는 코드: `app/data/lessonPrompt/` 로더 (미구현) → `app/api/lesson-plan`

---

## 0. 모든 파일 공통

| 항목 | 값 | 이유 |
|---|---|---|
| 인코딩 | **UTF-8 (BOM 없음)** | BOM 이 있으면 첫 열 이름이 `﻿교과` 가 된다 |
| 줄바꿈 | LF 또는 CRLF 둘 다 허용 | 로더가 `\r?\n` 으로 자른다 |
| 구분자 | 쉼표 `,` | |
| 따옴표 | 값에 쉼표·따옴표·줄바꿈이 있으면 `"` 로 감싼다. 내부 `"` 는 `""` | RFC 4180 |
| **줄바꿈 금지** | 셀 안에 줄바꿈을 넣지 않는다 | 성취기준 문장이 여러 행으로 쪼개지는 사고가 실제로 잦다 |
| 헤더 | **1행 필수, 아래 명시된 이름과 정확히 일치** | 순서는 바뀌어도 되고, 이름은 안 된다 |
| 빈 값 | 빈 문자열. `없음`·`N/A`·`-` 를 쓰지 않는다 | 로더가 "값 있음"으로 오인한다 |
| 모르는 값 | **비워 둔다. 추측해서 채우지 않는다** | 틀린 데이터가 빈 데이터보다 나쁘다 |

> ⚠️ **판정 근거를 길이나 존재 여부로 대체하지 않는다.** 파일이 있다고 정상이
> 아니고, 행이 많다고 완성이 아니다. 로더는 §5 검증을 통과한 행만 쓴다.

## 1. 성취기준 — `성취기준_<교과>.csv`

교과별 파일 하나씩. 예: `성취기준_국어.csv`

```csv
교과,학년군,영역,성취기준코드,성취기준
국어,5-6학년군,읽기,[6국02-03],글을 읽고 글쓴이가 말하고자 하는 주장이나 주제를 파악한다.
```

| 열 | 필수 | 형식 | 비고 |
|---|---|---|---|
| `교과` | ✔ | 아래 §6 교과명 | |
| `학년군` | ✔ | `1-2학년군` \| `3-4학년군` \| `5-6학년군` | |
| `영역` | ✔ | 자유 문자열 | 교육과정 문서의 영역명 그대로 |
| `성취기준코드` | ✔ | `[숫자][교과약칭][2자리]-[2자리]` | **대괄호 포함**. 예 `[6국02-03]` |
| `성취기준` | ✔ | 원문 그대로 | **한 글자도 바꾸지 않는다** |

**검증 규칙**
- 코드 앞 숫자(2/4/6)와 `학년군` 이 일치해야 한다 → 어긋나면 그 행 버림
- 같은 코드가 두 번 나오면 **둘 다 버리고** 로그에 남긴다 (어느 쪽이 맞는지 모름)
- 성취기준이 `.` 이나 `다` 로 끝나지 않으면 잘림 의심 → 경고

## 2. 단원 구성 — `단원구성.csv` ★핵심

사용자는 `[6국02-03]` 이 아니라 **"5학년 국어 4단원"** 으로 생각한다.
그 다리를 놓는 파일이고, 이게 있어야 "딸깍"이 성립한다.

```csv
교과,학년,학기,단원번호,단원명,총차시,차시번호,차시명,성취기준코드,출판사
국어,5,1,4,글쓴이의 주장,8,1,주장하는 글 살펴보기,[6국02-03],국정
국어,5,1,4,글쓴이의 주장,8,2,주장과 근거 파악하기,[6국02-03];[6국01-05],국정
```

| 열 | 필수 | 형식 | 비고 |
|---|---|---|---|
| `교과` | ✔ | §6 교과명 | |
| `학년` | ✔ | `1`~`6` | 숫자만 |
| `학기` | ✔ | `1` \| `2` | |
| `단원번호` | ✔ | 숫자 | 교과서 표기 기준 |
| `단원명` | ✔ | 2~40자 | **번호를 넣지 않는다** (`4단원 글쓴이의 주장` ✘ → `글쓴이의 주장` ✔) |
| `총차시` | ✔ | 숫자 | 그 단원 전체 차시 수 |
| `차시번호` | ✔ | 숫자, 1부터 | |
| `차시명` | ✔ | 2~40자 | |
| `성취기준코드` | | `[...]`, 여러 개면 `;` 로 연결 | **모르면 비워 둔다** |
| `출판사` | ✔ | `국정` 또는 출판사명 | 검정교과서는 출판사별로 행을 따로 |

**한 행 = 한 차시.** 단원 정보는 차시마다 반복된다(정규화하지 않는다) —
사람이 스프레드시트로 채우기 쉬운 형태를 우선한다.

**검증 규칙**
- `(교과, 학년, 학기, 단원번호, 차시번호, 출판사)` 가 유일해야 한다 → 중복은 첫 행만
- `차시번호 ≤ 총차시` → 어긋나면 그 행 버림
- 같은 단원 안에서 `단원명`·`총차시` 가 다르면 → 그 단원 전체를 경고 후 다수결
- `성취기준코드` 가 §1 에 없는 코드면 → **코드만 비우고 행은 살린다**
  (단원 정보 자체는 쓸모 있다)

## 3. 평가 기준 — `평가기준.csv`

```csv
교과,학년군,성취기준코드,평가요소,상,중,하
국어,5-6학년군,[6국02-03],주장과 근거 파악,주장과 근거를 모두 찾고 관계를 설명한다,주장과 근거를 찾는다,도움을 받아 주장을 찾는다
```

| 열 | 필수 | 비고 |
|---|---|---|
| `교과` `학년군` | ✔ | §1 과 같은 형식 |
| `성취기준코드` | ✔ | §1 에 존재해야 함 |
| `평가요소` | ✔ | 2~30자 |
| `상` `중` `하` | ✔ | 각 10~80자. **학생 행동으로 서술** ("~한다") |

## 4. 수업모형 전개 패턴 — `모형전개.csv`

지도안 "전개" 칸의 뼈대다. 실제 사례에서 뽑은 **교사 발화**가 핵심이다.

```csv
모형키,모형명,단계번호,단계명,교사발화예시,권장시간비율
direct,직접교수,1,설명하기,"오늘은 ~하는 방법을 배울 거예요. 먼저 ~가 무엇인지 알아봅시다.",20
direct,직접교수,2,시범보이기,"선생님이 먼저 해 볼게요. 잘 보세요.",18
```

| 열 | 필수 | 비고 |
|---|---|---|
| `모형키` | ✔ | **아래 §6 목록의 값만.** 새 모형은 임의로 만들지 않는다 |
| `모형명` | ✔ | 한글명 |
| `단계번호` | ✔ | 1부터 |
| `단계명` | ✔ | |
| `교사발화예시` | ✔ | **실제 지도안에서 발췌한 문장.** 지어내지 않는다 |
| `권장시간비율` | | 정수 %, 모형별 합 ≈ 100 | 비면 균등 배분 |

## 5. 로더가 하는 일 (구현 예정)

1. 파일이 **없으면 조용히 건너뛴다** — 데이터가 없어도 지도안 생성은 동작해야 한다
2. 행 단위 검증. **실패한 행만 버리고 줄번호와 이유를 남긴다**
   - 파싱 실패를 빈 배열·기본값으로 덮지 않는다
   - **탈락률 10% 초과 _그리고_ 탈락 5행 이상**이면 그 파일 전체를 쓰지 않는다
     - 둘 다 넘어야 하는 이유: 비율만 보면 3행 중 1행 틀렸을 때(33%) 멀쩡한
       2행까지 날아간다. 데이터는 조각조각 쌓이므로 초기 파일은 작다.
       반대로 건수만 보면 큰 파일에서 형식이 통째로 어긋난 걸 놓친다.
   - 열 이름이 하나라도 다르면 **행을 보지 않고 파일을 거부**한다
3. 통과한 데이터는 프롬프트에 **근거로만** 넣는다. 모델이 지어낸 코드보다
   이 표의 코드를 우선한다

## 6. 고정 어휘 (여기 없는 값을 쓰지 않는다)

**교과명** — `국어` `수학` `사회` `과학` `영어` `도덕` `체육` `음악` `미술`
`실과` `통합` `창의적체험활동`
(1~2학년은 `국어` `수학` `통합` `창의적체험활동` 만)

**모형키 / 모형명**

| 모형키 | 모형명 |
|---|---|
| `direct` | 직접교수 |
| `inquiry` | 탐구학습 |
| `cooperative` | 협동학습 |
| `problem` | 문제해결 |
| `discussion` | 토의토론 |
| `play` | 놀이중심 |
| `response` | 반응중심 |
| `value` | 가치갈등 |
| `project` | 프로젝트 |
| `concept` | 개념형성 |

원본은 `app/lib/lessonPlan.js` 의 `SUBJECTS` · `TEACHING_MODELS` 다.
**둘이 어긋나면 코드가 맞다** — 이 문서를 고친다.

## 7. 넣는 곳

```
snue-app/app/data/lessonPrompt/
  성취기준_국어.csv
  성취기준_수학.csv
  …
  단원구성.csv
  평가기준.csv
  모형전개.csv
```

파일을 넣고 `npm run lesson:check` 를 돌리면 몇 행이 통과·탈락했는지 나온다.
**탈락 행이 있으면 그걸 고치는 것이 다음 작업**이다.
