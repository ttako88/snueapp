# 포인트(SR) 경제 — 한 장 정리

> 소유자 확정: **내부의 모든 기능은 포인트로만 굴린다.** 리워드는 양도 불가·
> 플랫폼 내부용 (구글 리워드 광고 정책 전제).
>
> 화폐 분리 근거는 `docs/CURRENCY_SPLIT_DESIGN.md`. 여기서는 "얼마를 주고
> 얼마를 받는가" 만 다룬다.

---

## 1. 설계 원칙

1. **기여한 사람이 본다.** 강의평을 쓴 사람이 강의평을 읽는다. 이 앱의 데이터는
   사용자가 만들고, 무임승차가 쉬우면 아무도 안 쓴다.
2. **막히지는 않게 한다.** 포인트가 0이어도 앱이 벽돌이 되면 안 된다.
   광고 시청이라는 탈출구를 항상 열어 둔다.
3. **인플레를 만들지 않는다.** 획득이 소비보다 훨씬 크면 포인트가 무의미해진다.
   신규 유입 보너스는 크게, 반복 획득은 작게.
4. **돈으로 살 수 없다.** SR 을 팔면 리워드 광고 정책 전제가 깨진다(§CURRENCY §3).

---

## 2. 획득

| 행동 | SR | 제한 | 근거 |
|---|---|---|---|
| 학생 인증 완료 | +100 | 1회 | 011 `verification_bonus` |
| 강의평 작성(공개) | +20 | 과목·학기당 1회 | 011 `review_published` |
| 시험정보 작성 | +20 | 〃 | 011 `exam_tip_published` |
| 내 글이 도움됐어요 | +6 | 글당 상한 | 011 `helpful_bonus` |
| 리워드 광고 시청 | +5 | **하루 6회까지** | 미구현 |
| 실습 후기 작성 | +15 | 학기당 1회 | 미구현 |

**하루 광고 상한을 두는 이유**: 상한이 없으면 광고만 보는 계정이 생기고,
그 계정은 기여 없이 모든 걸 열람한다. 원칙 1이 무너진다.

## 3. 소비

| 기능 | SR | 근거 |
|---|---|---|
| 강의평 열람(과목 1개, 30일) | −5 | 011 `unlock_subject` |
| 지도안 약안 생성 | −10 | 원가 ₩22 → §4 참조 |
| 지도안 세안 생성 | −25 | 원가 ₩40~50 추정 |
| 실습학교 게시판 | **무료** | 인증이 이미 장벽 |
| 급식·시간표·강의검색 | **무료** | 정보 축은 열어 둔다 |

## 4. 지도안 값을 이렇게 잡은 이유

실측 원가(2026-07-22): 약안 1건 **₩22** (프롬프트 v2 기준).

- 광고 1회 시청 = +5 SR. 약안 1건 = −10 SR → **광고 2회면 약안 하나.**
- 광고 eCPM 을 보수적으로 잡아도(국내 리워드 ₩3,000~8,000/1,000회 추정)
  2회 시청이면 ₩6~16 이다. 원가 ₩22 를 **다 못 덮는다.**
- 즉 지도안은 **적자 기능이다.** 그래도 넣는 이유는 이게 유입 동기이기 때문이다.
  다만 무제한으로 두면 안 된다 → 일일 예산 상한(018)이 이미 있다.

> ⚠️ eCPM 은 추정치다. 실제 광고를 붙여 보기 전에는 모른다.
> 붙인 뒤 첫 달 실적으로 이 표를 다시 계산해야 한다.

## 5. 신규 사용자 여정 (막히는지 검증)

```
가입 → 학생 인증  +100
  ↓
강의평 3과목 열람  −15   (잔액 85)
  ↓
지도안 약안 3건    −30   (잔액 55)
  ↓
강의평 2개 작성    +40   (잔액 95)
```

인증만 해도 **한동안 안 막힌다.** 인증을 안 하면 시작 잔액이 0이라
광고를 봐야 하는데, 이건 의도한 압력이다 — 인증이 커뮤니티 품질의 근간이다.

## 6. 아직 안 정한 것

- **SR 만료.** 없으면 계속 쌓이고, 있으면 사용자가 화낸다.
  절충안: 기여 보상은 만료 없음, **광고 리워드만 90일**. 원장의 `reason` 으로
  구분 가능하므로 구현은 된다. 정책 판단이 필요하다.
- **포인트 교환소**(카톡 대화에 나온 아이디어). 양도 불가 원칙과 충돌하므로
  "사용자끼리 주고받기" 는 불가. **플랫폼이 주는 것**만 가능하다
  (예: SR → 광고 제거 3일). 이건 정책 위반이 아니다.
- **파워링크권**(홍보게시판 상단 노출). SR 로만 사면 정책상 문제없다.
  다만 돈으로 사게 하는 순간 광고 상품이 되어 사업자등록 이슈가 붙는다.

## 7. 구현 상태

| 항목 | 상태 |
|---|---|
| 원장 (`ticket_ledger`) | 운영 적용됨 (011) |
| 화폐 분리 (`currency`) | `pending/022` — 승인 대기 |
| 인증 보너스 | 운영 적용됨 |
| 강의평 보상·열람 차감 | `pending/016` — 승인 대기, flag OFF |
| 지도안 SR 차감 | **미구현** (지금은 예산 상한만 있고 SR 을 안 깎는다) |
| 리워드 광고 | 미구현 |

**가장 먼저 붙일 것**: 지도안 SR 차감. 지금은 로그인만 하면 무제한이라
(일일 예산 상한에 걸릴 때까지) 소유자 지갑이 그대로 열려 있다.
