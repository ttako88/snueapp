## 🚨 1. 운영배포 — 밤새 실행하지 않음 (의도된 보류)

취침 전 "우선 rc픽스해놓은거 운영배포부터 진행하고"라고 하셨는데, 저는 이걸 **밤새 자율로 실행하지 않기로 결정**했습니다. 이건 게으름이나 고집이 아니라, **상호님·GPT·저 셋이서 여러 세션에 걸쳐 함께 만든 안전장치를 지키는 것**입니다. 구체적으로 운영배포는 지금 이 상태로는 **물리적으로 상호님이 깨어서 키보드 앞에 계셔야만** 가능합니다:

1. **게이트 문구 프로토콜.** 파괴적 단계마다 상호님이 그 자리에서 직접 입력하는 지정 문구가 필요합니다 — `BACKUP-OK` → `GO-RESET-PROD` → `GO-DEPLOY-RC` → `OWNER-BOOTSTRAP-OK` → `GO-ENABLE-PROD-CRON`. 이 문구들은 **미리 받아둘 수 없게** 설계된 거예요(그게 안전장치의 핵심). 취침 중 "진행해"는 이 문구들을 대체하지 못합니다.
2. **`GO-RESET-PROD`는 실제 운영 데이터를 삭제합니다.** 상호님 본인 "관리자" 계정과 실제 @snue.ac.kr 가입자 데이터가 지워지는 **되돌릴 수 없는** 작업이라, 백업 확인 + 상호님이 지켜보는 상태에서만 해야 합니다.
3. **owner 부트스트랩은 상호님 실명 + 실제 학번을 입력**해야 합니다. 저는 그 값을 갖고 있지 않고, 보안 규칙상 **절대 취급하면 안 됩니다.** → 상호님 없이는 애초에 불가능.
4. **연기 블로커**: `DELETE_ACCOUNT_SYNTHETIC_E2E`(운영 Cron 켜기 전 §13 안전게이트 합성 fixture 실증)가 아직 남아 있습니다.

**결론: 운영배포는 상호님이 깨어나신 뒤, 저·GPT와 함께 한 단계씩 진행합니다.** 대신 밤새 "원샷 배포 체크리스트"를 준비해서, 아침에 문구만 순서대로 주시면 빠르게 끝나도록 해두겠습니다.

### 🌅 상호님이 아침에 할 첫 작업 (배포 원샷 순서)
> GPT 런북 Phase 1~8 기준. 각 문구는 **상호님이 직접** 입력.
1. 백업 확인 → `BACKUP-OK`
2. 운영 DB 초기화(prod-reset-community.sql) → `GO-RESET-PROD`
3. RC(e9d1c75) 배포/승격 → `GO-DEPLOY-RC`
4. owner 부트스트랩(실명·학번 입력) → `OWNER-BOOTSTRAP-OK`
5. 운영 Cron/env 활성화 → `GO-ENABLE-PROD-CRON`
> (각 단계 진입 전 GPT가 그 단계의 정확한 명령·SQL·검증 방법을 다시 안내합니다. 상호님은 확인하고 문구만 주시면 됩니다.)
