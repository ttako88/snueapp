## 0. 이 지시서의 최상위 규칙

소유자의 대전제:
> "그저 내가 아이디어를 내고 잠들면, 아침엔 그게 구현이 돼서 적용되어있는걸 바랄뿐이야"

여기서 따라오는 것들:

1. **멈추지 않는다.** 어떤 항목이 막히면 그 항목만 `pending` 으로 걸고
   **즉시 다음 항목으로 간다.** 07-21 에 GPT 회신을 기다리며 한 시간 만에
   멈춰 서서 지적받았다. 같은 실수를 반복하지 않는다.
2. **소유자를 부르지 않는다.** L3(운영 mutation·시크릿·삭제·flag ON)를 만나면
   깨우지 말고 pending. 승인 필요 항목은 아침 보고서 맨 위에 한 줄씩 모아 둔다.
3. **완료라고 쓰지 않는다.** `CODE_WRITTEN` / `LOCAL_VERIFIED` /
   `PROD_VERIFIED` / `USER_REACHABLE` 중 하나로 적는다.
4. **판단이 필요하면 GPT 에게 던지고 답을 기다리는 동안 다른 일을 한다.**
   회신 대기는 블로커가 아니다.

### 종료 조건
KST 2026-07-22 **12:00**. 그 전에 할 일이 떨어지면 §9 무한 백로그로 간다.
"할 일이 없다"는 이 지시서에서 성립하지 않는다.

## 1. 최우선 — 지도안 "딸깍" 옵션 설계 (소유자 명시 지시)

> "너는 약안 세안에 사용자가 어떤 옵션을 설정하면 진짜 딸깍 하고 양질의
> 약안 세안이 뽑혀나올지 지피티랑 연구를 해서, 로컬 gpt가 정리를 해주면
> 바로 구현할수 있도록 미리 준비를 해놓자. (…) 나 자는사이에 해야지."

이게 오늘 밤 1순위다. 로컬 GPT 가 2차 지시서대로 단원구성 CSV 를 만들어 오면
**그날 바로 붙일 수 있는 상태**여야 한다.

### 1-1. GPT 에게 던질 질문 (즉시)
- 사용자가 실제로 고를 수 있는 최소 옵션 집합은? (많으면 안 쓴다, 적으면 품질이 안 나온다)
- 옵션 중 **자동 유도 가능한 것**은 무엇인가 (학년+교과+단원 → 성취기준·차시목표)
- 교생이 실제로 지도교사에게 지적받는 항목 top 10 → 그걸 프롬프트 제약으로 환산
- 약안/세안 프롬프트를 하나로 갈지 분리할지

### 1-2. 내가 병렬로 진행할 것
- `app/lib/lessonPlan.js` 의 옵션 스키마를 **2단계**로 재설계
  - **1단계(필수 4개):** 학년 / 교과 / 단원·주제 / 차시 시간
  - **2단계(선택, 접힘):** 수업모형 · 학습자 특성 · 중점 활동 · 평가 방식 ·
    교실 환경(기자재) · 지도교사 요구사항 자유입력
- 옵션 → 프롬프트 조립기를 **데이터 주도**로 바꾼다. 지금은 문자열이 코드에
  박혀 있어 로컬 GPT 산출물을 붙일 자리가 없다.
  - `app/data/lessonPrompt/` 신설: `subjectHints.json`, `modelSteps.json`,
    `gradeVoice.json`, `evalRubric.json`
  - **파일이 비어 있어도 동작**해야 한다 (있으면 품질이 올라가는 구조)
- `단원구성.csv` 를 삼킬 로더의 **스키마를 먼저 확정**해서 문서화
  (`docs/LESSON_DATA_CONTRACT.md`) — 로컬 GPT 가 형식을 어긋나게 만들면
  전부 다시 해야 하므로 계약을 먼저 고정한다
- 프롬프트 A/B 를 돌릴 수 있게 `gen-lesson-samples.mjs` 에 `--prompt <버전>` 추가

### 1-3. 키가 들어오면
`npm run lesson:samples` 실행 → 바탕화면 `지도안_출력물/` 에 10건 저장 →
아침 보고서에 폴더 경로와 총 원가를 적는다. 키가 없으면 이 항목만 건너뛴다.

## 2. 대기 중인 마이그레이션 정리

### 2-1. 019 practicum placement
- GPT 검수 요청 (이미 dry-run PASS)
- 검수 회신 전에 **앱 쪽 화면·API 를 먼저 만든다** (flag OFF)
  - 학기별 실습학교 설정 UI (2-1 ~ 4-1 다섯 칸)
  - `app/data/practicumSchools.json` 과 연결
  - 배정 안 한 학기는 게시판 진입 시 설정 유도
- 회신 오면 반영 → **적용은 pending** (운영 스키마 = L3)

### 2-2. 016 강의평가
- 과목 마스터가 동결이라 **적용은 못 한다.** 대신:
  - 미해결 결함 2건(동시 finalize claim 전이 / 011 항목별 denominator)을
    **SQL 로 고쳐서 016 에 반영**한다 — 이건 파일 수정이라 승인 불필요
  - 고친 뒤 dry-run 재실행으로 회귀 없음 확인
  - GPT 재검수 요청

### 2-3. 015 도메인 게이트 폐기
낮에만. 밤에는 건드리지 않는다. **파일은 손대지 말 것.**

## 3. 미해결 결함 소각

| 결함 | 오늘 밤 할 일 |

| 동시 finalize claim 전이 | 재현 스크립트 작성 → 실패 확인 → 016 에서 수정 → 재현 스크립트 PASS |
