## 2. 분석·수익화 (코드·migration 배포됨, 활성화 대기)
- ⏸️ **productAnalytics 대시보드 켜기** — `/admin/analytics`. 사업자 불요. 켜기 전 **동의 UI·처리방침 문구 확정** 필요(법적 문구).
- ⏸️ **targetedAds(맞춤광고)** — 전부 구현·휴면. **사업자등록 완료 + app_flags.targeted_ads=true** 후 켬. 광고주엔 개인정보 미전송·집계만.

## 7.5 관리자 데이터 업로드·자동적재 파이프라인 (2026-07-23 소유자 지시, 신규·중요)
> "학기마다 명단·시간표 공지 새로 올라오면 DB 업데이트 해줘야 하는 구조. **파일 하나 주면 알아서
> 파싱돼서 DB로 들어가는 프로그램을 패키지로 같이 만들어라.** 손으로 안 올려도 되게."
- 🔜 **실습학교 명단 업로드기** — HWP/xlsx/텍스트 원본 업로드 → 파싱(학교·지역·실습종류·인원) → **검수 미리보기** → 확정 시 practicumSchools/ school_catalog 반영. 학기 필드 필수. (첫 사용: 2026-1 신용산초 포함 정식목록 — 소유자 제공)
- 🔜 **수강시간표(강의 데이터) 업로드기** — 학교 xlsx(현 파서: HTTP응답째 저장→PK시그니처 잘라 8시트 파싱, 심화과정 forward-fill) 자동화. 업로드→분류(학년·심화·교직·교양)→courses-YYYY-N.json/DB. 현재는 수동 스크립트.
- 설계 원칙: **재사용 파서 + 검수 단계(자동추출 그대로 믿지 말 것 — 신용산초 누락·"강남서초" 오인 전례)** + 학기 버전관리 + 관리자(/admin) 전용 + 롤백(이전 버전 보존). GPT 검수 후 flag 뒤 구현.
