# 네이티브 앱 출시 준비

> 소유자 질문: *"어플출시를 서두르는 방법은?"*
>
> 결론부터: **웹앱을 Capacitor 로 감싸는 것이 가장 빠릅니다.** 지금 코드를
> 거의 그대로 씁니다. 다만 "빠르다" 는 개발 기간 얘기고, **심사와 계정 준비에
> 걸리는 시간은 줄일 수 없습니다.** 그쪽을 먼저 시작하는 게 실제로 앞당기는
> 유일한 방법입니다.

## 1. 실제 병목은 개발이 아니다

| 항목 | 소요 | 지금 시작 가능? |
|---|---|---|
| Apple Developer Program 가입 | 심사 며칠 + **연 $99** | ✔ 지금 |
| Google Play 개발자 등록 | **$25 1회** + 신원확인 | ✔ 지금 |
| Play 신규 개인 개발자 **비공개 테스트** | **테스터 12명 · 14일 연속** | ✔ 지금 |
| 개인정보처리방침 URL | 반나절 | ✔ 지금 |
| Capacitor 래핑 | 1~2일 | 코드 준비됨 |
| 스토어 심사 | 각 1~7일 | 위가 끝나야 |

### ⚠️ Google Play 개인 계정 요건이 크리티컬 패스입니다 (2026-07-22 확인)

**2023-11-13 이후에 만든 개인(personal) 계정**은 정식 출시 전에 비공개 테스트를
**테스터 12명 이상 · 14일 연속 옵트인** 으로 돌려야 합니다.
- 원래 20명이었는데 2024-12-11 에 12명으로 완화됐습니다.
- 2025년~2026년 상반기 중 추가 변경 없음.

### ★ 그런데 사업자등록이 있으면 이 요건이 면제됩니다

**법인·사업자로 등록한 organization 계정은 12명/14일 요건에서 제외**됩니다.

이게 중요한 이유는 `docs/CURRENCY_SPLIT_DESIGN.md` 와 이어지기 때문입니다.
사업자등록은 **유료 기능을 열기 위해서도 필요**한데, 같은 등록 하나가
**출시를 2주 앞당기는 효과**도 냅니다.

| | 개인 계정 | 사업자 계정 |
|---|---|---|
| 테스터 12명·14일 | **필요** | 면제 |
| 유료 크레딧 판매 | 불가 | 가능 |
| 앱 내 결제 | 사실상 불가 | 가능 |

→ **사업자등록을 먼저 하는 쪽이 여러 문제를 한 번에 풉니다.**
다만 등록에는 세무 의무가 따라오므로 이건 순수하게 소유자 판단입니다.
(저는 세무 조언을 할 수 없습니다.)

**그래서 오늘 할 수 있는 가장 효과적인 일**
- 사업자등록을 할지 먼저 결정 → 하면 개인 계정 경로를 아예 건너뜁니다
- 개인으로 간다면 **테스터 12명 명단부터** 확보 (개발보다 먼저)
- 어느 쪽이든 개발자 계정 등록·결제는 제가 대신 못 합니다.

## 2. Capacitor 래핑

### 왜 Capacitor 인가
- 지금 Next.js 코드를 그대로 감쌉니다. React Native 로 다시 짜지 않습니다.
- 나중에 위젯·푸시 같은 네이티브 기능을 플러그인으로 붙일 수 있습니다.

### ⚠️ 가장 흔한 사고: `server.url`
`capacitor.config` 의 `server.url` 로 배포된 웹사이트를 가리키면 **개발 중에는**
편합니다. 하지만 **출시 빌드에 그대로 두면 안 됩니다.**
- 앱이 껍데기만 있고 내용은 전부 원격이 되어, 스토어가 "웹뷰 래퍼" 로 보고
  거부할 수 있습니다 (특히 Apple Guideline 4.2 최소 기능).
- 서버가 죽으면 앱이 백지가 됩니다.

→ **정적 산출물을 앱 안에 넣고(`output: "export"` 또는 정적 내보내기), API 만
원격을 부릅니다.** 지금 앱은 서버 라우트를 쓰므로 이 부분 설계가 필요합니다
(§4 참조).

### 순서
1. `@capacitor/core` `@capacitor/cli` 설치 · `npx cap init`
2. 웹 자산 빌드 → `npx cap add ios android`
3. 아이콘·스플래시 생성
4. 실기기 확인 → 스토어 업로드

## 3. 준비물

### 아이콘·스플래시
- 원본 **1024×1024 PNG**(투명 없음) 하나면 나머지는 도구가 만듭니다.
- 지금 `app/icon.png` 가 있으니 해상도만 확인하면 됩니다.

### 번들 ID (한 번 정하면 못 바꿉니다)
- 제안: `kr.ac.snue.saerok` 또는 `app.saerok.snue`
- ⚠️ `ac.kr` 도메인은 학교 소유입니다. 학교와 무관한 개인 앱이면
  **학교 도메인을 쓰지 않는 편이 안전**합니다 → `app.saerok.snue` 쪽 권장.

### 스토어 등록물
- 스크린샷: iPhone 6.7"·6.1" / Android 폰 각 2~8장
- 짧은 설명(80자) · 긴 설명(4000자) · 키워드
- **개인정보처리방침 URL 필수** (아래)
- 데이터 수집 신고: Apple "앱 개인정보 보호", Google "데이터 보안"
  → 이 앱은 **이메일·학번 해시·재학증명서**를 다룹니다. 정확히 신고해야 합니다.

## 4. 이 앱에만 해당하는 걸림돌

### 4-1. 서버 라우트 의존
`/api/lesson-plan`, `/api/verification/*` 등은 Next 서버가 필요합니다.
정적 내보내기를 하면 이 라우트들이 사라집니다.

→ **앱은 정적 자산 + Vercel API 호출** 구조로 갑니다. 즉 API 는 지금처럼
Vercel 에 두고, 앱은 그걸 부르는 클라이언트가 됩니다. 이때:
- API 에 CORS 허용 오리진을 앱 스킴으로 열어야 합니다
- 인증 토큰을 앱 저장소에 안전하게 보관해야 합니다(`@capacitor/preferences`
  는 평문입니다 — 토큰은 Keychain/Keystore 플러그인을 씁니다)

### 4-2. 재학증명서 업로드
카메라·파일 접근 권한이 필요합니다. 권한 설명 문구(`NSCameraUsageDescription`
등)를 **한국어로** 정확히 써야 심사에서 안 막힙니다.

### 4-3. 리워드 광고
AdMob 플러그인이 필요하고, 앱 심사와 별개로 AdMob 계정 승인이 있습니다.
**출시 1차에서는 빼는 것을 권합니다** — 광고가 있으면 심사가 까다로워지고,
초기엔 트래픽이 없어 수익도 없습니다.

### 4-4. 홈스크린 위젯 (소유자 희망)
- iOS: WidgetKit(Swift), Android: Glance/RemoteViews — **네이티브 코드가 필요합니다.**
  Capacitor 로 감싸는 것과 별개 작업입니다.
- 위젯에 띄울 후보: **오늘 급식** / 다음 수업 / 실습 D-day
  → 급식이 가장 적합합니다. 하루 한 번 바뀌고, 로그인 없이도 의미가 있고,
    한 줄로 요약됩니다.
- **1차 출시에는 넣지 말 것.** 위젯은 별도 데이터 갱신 경로가 필요해서
  출시를 늦춥니다. 2차에서.

## 5. 권장 순서

```
지금        개발자 계정 2개 등록 + 테스터 12명 모으기   ← 소유자만 가능
그동안     개인정보처리방침 작성 · API CORS · 토큰 저장 정리
그다음     Capacitor 래핑 · 아이콘 · 스크린샷
마지막     비공개 테스트 14일 → 정식 심사
```

**1번이 안 끝나면 나머지를 아무리 빨리 해도 출시가 안 됩니다.**

## 6. 제가 대신 못 하는 것

- 개발자 계정 생성·결제 (계정 생성과 결제는 제가 하지 않습니다)
- 스토어 콘솔 로그인
- 테스터 모집

나머지(코드·문서·설정 파일)는 제가 준비해 둘 수 있습니다.
