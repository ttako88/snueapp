## 확정 로드맵 (사용자, 2026-07-21)
광고 삽입 + 유료 결제 + 크레딧 운용이 정식 로드맵. 크레딧은 약안/세안 AI 툴의 **LLM 사용량 미터링**용. **진짜 비용 축 = LLM API**, 호스팅 아님.

## 설계 상수 (구현 전 미리 확정 — GPT 검수)

| 영역 | 제약 |
|---|---|
| 용어·장부 분리 | 1차 활동보상은 **`reward_points`**로 명명. 미래 구매형 크레딧·결제잔액과 **절대 같은 장부 재사용 금지** |
| 포인트 원장 | 행 **불변**(수정·삭제 금지). 지급/차감/취소를 새 행으로. `source_type/source_id/idempotency_key` UNIQUE로 중복 지급·차감 차단 |
| 잔액 | 원장 합계가 최종 기준. 캐시 잔액을 둬도 원장과 **같은 DB 트랜잭션**에서만 변경, 클라이언트 직접 UPDATE 금지 |
| LLM 미터링 | 호출 전 예상 포인트 **reserve** → 성공 후 실제 토큰으로 **settle** → 실패·시간초과 시 **release**. 동시 호출로 잔액 음수 방지(행 잠금 또는 원자 RPC) |
| 비용 기록 | model/provider/model_version/input·output_tokens/provider_request_id/cost_snapshot/**rate_version**. 환산율 버전 관리로 모델 가격 변경 후에도 과거 차감 근거 유지 |
| 비용 안전장치 | 사용자별 일·월 상한, 전역 월 예산, 모델별 상한, **비상 kill switch**를 서버 집행. LLM 키·원가 클라이언트 비노출 |
| 민감정보 | 학생증·실명·학번·제재 신원·관리자 사건자료는 **어떤 LLM 요청에도 미전달**. 프롬프트·응답 원문 장기저장 기본 OFF |
| 커뮤니티 독립성 | 포인트로 **제재 해제·신고 우선순위·운영권한·게시물 노출 구매 불가**. 포인트 부족은 약안/세안 도구만 제한, 일반 커뮤니티 권한과 분리 |
| 운영자 조정 | 수동 지급·회수도 전용 RPC+사유코드+감사로그. DB 직접 수정은 break-glass만 |
| 광고 | 로그인·학생인증·운영콘솔·신원조회 화면엔 **광고·제3자 추적 금지**. 커뮤니티 화면도 CSP allowlist·처리방침·동의 확정 후 활성화 |
| 결제 | 브라우저 성공 redirect 불신, **서명 검증된 PG webhook이 결제 상태 최종 기준**. webhook event_id UNIQUE, 환불·취소·차지백은 원장 역분개 |

## 규제 문구 (정정 — GPT)
"1차 활동보상 포인트라서 선불전자지급수단·세무 규제를 **회피한다**"로 단정하지 않는다. →
"구매·양도·환급·앱외 사용을 막아 **해당 규제의 적용 가능성과 이용자 위험을 낮추되, 최종 판단은 출시 전 법률·세무 검토에서 확정한다.**"
