# 일회성 부트스트랩 — 앞으로 소유자를 부르지 않기 위해 지금 받는 것

> 상태: **초안.** GPT 검수(C-20260722-ONE_SHOT_BOOTSTRAP_COMPLETENESS) 회신 후 확정.
> 목적: 소유자가 "마지막으로 한 번만" 고생하고 다시는 불려나오지 않게 하는 것.
>
> 경계 재정의 (2026-07-22):
> 기준은 "Claude 가 자격증명을 드느냐" 가 아니다 — 이미 `PROD_DB_URL` 로
> postgres 전권을 들고 있으므로 그 기준은 이미 무너져 있다.
> 진짜 기준은 **소유자의 개인 인증수단인가, 자동화용 기계 토큰인가** 다.
> 기계 토큰은 범위를 좁힐 수 있고 개별 폐기가 가능하며 소유자 계정과 분리된다.
> postgres 전권보다 오히려 안전하다.

## B. 인프라 기계 토큰 — 이게 호출을 없애는 핵심

| # | 값 | 이게 없으면 소유자를 부르는 상황 | 발급처 | 최소 범위 |
|---|---|---|---|---|
| B1 | `SUPABASE_SECRET_KEY` | 인증 라우트 전부 503 | Supabase → Settings → API Keys | 프로젝트 단위(고정) |
| B2 | `SUPABASE_ACCESS_TOKEN` (Management API) | 소셜로그인 키 입력, 메일 템플릿, rate limit, Auth 설정 변경 | supabase.com/dashboard/account/tokens | 이 프로젝트만 |
| B3 | `VERCEL_TOKEN` | **새 env 생길 때마다**, 배포, 롤백 | vercel.com/account/tokens | Scope = 이 프로젝트 |
| B4 | `VERCEL_PROJECT_ID` / `VERCEL_ORG_ID` | 위와 동일 | Vercel → Project → Settings | 비밀 아님 |
| B5 | `GITHUB_TOKEN` (fine-grained PAT) | 푸시·PR·릴리스 | github.com/settings/tokens | 이 저장소만 · Contents RW + PR RW |

> B3 가 가장 큰 항목이다. 지금 구조에서는 env 하나 늘 때마다 소유자가
> Vercel 대시보드를 열어야 한다. 토큰이 있으면 그게 전부 사라진다.

### D3. 모더레이션 위임 — 소유자 계정 로그인은 필요 없다

신고 처리·게시물 삭제를 위임받으려면 `moderator` 이상 권한이 필요하다
(`actor_role_check` 가 `auth.uid()` 를 본다 — service_role 만으로는 안 된다).

**소유자 계정으로 로그인하지 않는다.** 대신 전용 봇 계정을 쓴다.

- `SUPABASE_SECRET_KEY` 로 봇 auth 계정을 **내가 생성**한다
- `private.members` 에서 그 계정에 `moderator` 역할 부여
- 필요할 때 내가 그 계정 세션을 발급해 조치를 수행

이유:
- **감사 로그에 "봇" 으로 남는다.** 실제 학생 글을 지우는 일이므로
  소유자 개인 명의로 기록되지 않는 편이 옳다
- 문제 시 봇 권한만 내리면 된다. 소유자 계정은 건드리지 않는다
- 소유자 비밀번호·2FA 가 **영원히 불필요**

→ 소유자가 줄 것: 없음. 내가 만든다.

## E. 비밀이 아닌 결정사항 — 이것도 미리 받아야 안 부른다

값이 아니라 **판단**이라서, 안 받아두면 나중에 반드시 묻게 된다.

| # | 항목 | 기본값(소유자가 안 정하면 이걸로 간다) |
|---|---|---|
| E1 | 앱에 표시할 문의 이메일 | 소유자 개인 메일 |
| E2 | 신고 처리 기준 | 명백한 욕설·개인정보 노출·불법은 즉시 숨김, 나머지는 보류 후 아침 보고 |
| E3 | 야간 배포 시간대 | 제한 없음 |
| E4 | 롤백 판단 권한 | 내가 판단해 실행하고 아침에 보고 |
| E5 | 새 모듈 추가 판단 | 내가 판단. 아침 보고 후 "필요 없다" 하시면 제거 |
| E6 | 개인정보처리방침 연락처 | E1 과 동일 |

## G. 다 받아도 남는 것 — 물리적으로 위임 불가

1. **결제·과금** — 광고 수익화, 유료 플랜 전환
2. **앱스토어 등록** — 네이티브 전환 시 개발자 계정 명의
3. **법적 문서 서명** — 약관·처리방침의 최종 책임
4. **각 토큰의 최초 발급** — 계정 로그인·2FA 가 필요한 그 한 번

> 4번이 이번 마법사에서 소유자가 하는 유일한 작업이다.
> 토큰 발급 화면에서 값을 복사해 마법사에 붙여넣는 것,
> 그 이후로는 부르지 않는다.

## H. 폭발반경 완화 (소유자 개입 없이 유지)

토큰을 여럿 들면 유출 시 피해가 커진다. 소유자가 관리하지 않아도 되게 설계한다.

- **보관**: `.env.prod.local` (git 비추적, 0600). 값은 로그·argv·채팅에 출력 금지
- **범위**: 발급 시점에 최소 범위로 고정 (프로젝트 단위, 저장소 단위)
- **폐기**: 각 토큰이 어디서 폐기되는지 이 문서에 경로를 적어둔다.
  소유자는 사고 시 그 링크만 누르면 된다
- **분리**: 토큰 하나가 유출돼도 다른 것으로 번지지 않게 서로 다른 provider
- **탐지**: 아침 보고서에 각 토큰의 최근 사용 여부를 비밀 아닌 형태로 기록
- **정기 교체**: 하지 않는다. 소유자 개입이 필요해지므로,
  대신 범위를 좁히고 폐기 경로를 문서화하는 쪽을 택한다
